# BACKEND DEPLOYMENT GUIDE - SK AGROVET

## 🚀 Quick Deployment Options

### **RECOMMENDED: Railway.app (Fastest)**
**Why Railway:** Free tier available, PostgreSQL included, simple setup, auto-deploys from GitHub

---

## 📋 Step 1: Prepare Backend for Deployment

### Check Backend is Build-Ready
```bash
cd backend
npm run build
```
✅ If no errors, you're ready!

---

## 🚂 Step 2: Deploy to Railway.app

### A. Sign Up
1. Go to https://railway.app
2. Click "Login with GitHub" 
3. Authorize Railway to access GitHub

### B. Create New Project
1. Click "Create New Project"
2. Select "Deploy from GitHub repo"
3. Select your repository: `Agrovet-Web-Based-System`
4. Select branch: `main`

### C. Configure Build Settings
1. Railway auto-detects `backend/` folder
2. **Build Command:** `npm run build`
3. **Start Command:** `npm start`
4. **Root Directory:** `backend`

### D. Add PostgreSQL Database
1. In Railway dashboard, click "Add Service"
2. Select "PostgreSQL"
3. Railway automatically provides connection string

### E. Set Environment Variables
In Railway project settings, add:

```
DATABASE_URL=<auto-provided by Railway>
DB_HOST=<provided by Railway>
DB_PORT=5432
DB_NAME=sk_agrovet
DB_USER=postgres
DB_PASSWORD=<generated by Railway>

JWT_SECRET=your_super_secret_jwt_key_change_this
JWT_EXPIRE=7d

NODE_ENV=production
PORT=8000

CORS_ORIGIN=https://skagro.netlify.app

# Email (optional for now)
EMAIL_USER=your.email@gmail.com
EMAIL_PASS=your_app_password
```

### F. Get Your Backend URL
After deployment completes:
- Railway shows your app URL (example: `https://agrovet-api.railway.app`)
- Copy this URL

---

## 🌐 Step 3: Update Netlify Frontend with Backend URL

### A. Go to Netlify Dashboard
1. https://app.netlify.com
2. Select your site: `skagro`

### B. Add Environment Variable
1. Go to **Site settings** → **Build & deploy** → **Environment**
2. Click **Add environment variable**
3. Add:
   ```
   VITE_API_URL=https://your-railway-backend-url/api
   ```
   (Example: `https://agrovet-api.railway.app/api`)

### C. Trigger Redeployment
1. Go to **Deploys** tab
2. Click **Trigger deploy** → **Deploy site**
3. Wait for build to complete
4. Check build logs for success

---

## ✅ Verify Deployment Works

### Test API Connection:
```bash
# Replace with your Railway backend URL
curl https://your-railway-url/api/health

# Should return:
# {"status":"ok","timestamp":"...","environment":"production"}
```

### Test Frontend:
1. Go to https://skagro.netlify.app/login
2. Try to login
3. Should work without network error!

---

## 🆘 Troubleshooting

### If Login Still Gives Network Error:

1. **Check VITE_API_URL is Set:**
   - Netlify → Site settings → Environment
   - Verify variable is there with correct URL

2. **Redeploy Frontend:**
   - Go to Netlify Deploys tab
   - Click "Trigger deploy" → "Deploy site"

3. **Check Backend is Running:**
   ```bash
   curl https://your-backend-url/api/health
   ```
   Should not be Connection Refused

4. **Check CORS:**
   - Netlify domain must be in backend CORS_ORIGIN
   - Verify: `CORS_ORIGIN=https://skagro.netlify.app`

5. **Browser Console:**
   - Open DevTools (F12) → Console tab
   - Look for actual error message
   - Screenshot and share if stuck

## 🔧 Alternative: Deploy to Render.com

If Railway has issues:

### D. Sign Up
1. https://render.com
2. Click "Sign up"
3. Choose "GitHub"

### B. Create Web Service
1. Click "New+" → "Web Service"
2. Connect GitHub repo
3. Settings:
   - **Name:** sk-agrovet-api
   - **Environment:** Node
   - **Build Command:** `cd backend && npm install && npm run build`
   - **Start Command:** `cd backend && npm start`
   - **Root Directory:** `.` (leave as is)

### C. Environment Variables
Add same variables as Railway section above

### D. Get URL
- Render provides URL after deployment
- Add to Netlify as `VITE_API_URL`
